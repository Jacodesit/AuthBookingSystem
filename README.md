Simple Booking System with Authentication

This project is a simple booking system where users must create an account to make a booking.
Each user can only have one active booking per date.
The system prevents double bookings and keeps booking data organized and clear. This project focuses on real-world logic.
It practices authentication, validation, and rule-based booking flow.

Features
• User registration
• User login and logout
• Create a booking
• View own bookings
• Cancel a booking
• Prevent multiple active bookings on the same date
• Booking status handling
• Clean navigation between pages

Booking Rules
• User must be logged in to book
• One active booking per user per date
• Cancelled bookings free the date
• Past dates cannot be booked

Tech Stack
• Laravel
• Inertia.js
• React
• TypeScript
• Tailwind CSS
• MySQL
